Lab Pods With YAML

1)
Create a YAML file at /root/web-pod.yaml with the following specifications:

    Pod Name: web-pod
    Container Name: nginx-container
    Image: nginx
    Labels:
        app: web
        env: prod

Apply the YAML file to create the Pod.

Solution:
Utilize the YAML file provided below to create a Pod and save it in the specified location: /root/web-pod.yaml.

apiVersion: v1
kind: Pod
metadata:
  name: web-pod
  labels:
    app: web
    env: prod
spec:
  containers:
    - name: nginx-container
      image: nginx

Once the YAML file is created, execute the following command to create the Pod:

kubectl apply -f /root/web-pod.yaml

4)
You have been provided with the following YAML file located at /root/broken-pod.yaml. This file is intended to create a Pod named static-web using the httpd image. 
However, the file contains multiple issues. Fix the issues and deploy the pod.

The file contains multiple issues, so your task is to identify and resolve these issues and then deploy the Pod successfully.

Solution:
Utilize the fixed YAML file provided below:

apiVersion: v1
kind: Pod
metadata:
  name: static-web
  labels:
    app: static
    env: prod
spec:
  containers:
    - name: httpd-container
      image: httpd

Once the YAML file is created, execute the following command to create the Pod:

kubectl apply -f /root/broken-pod.yaml


5)
A Pod named node-api has been deployed. The container within this Pod contains an environment variable named APP_COLOR.

Identify the value of the APP_COLOR environment variable configured inside the container.

Solution:
Use the following command to describe the Pod and view the environment variables configured in the container spec:

kubectl describe pod node-api

Look under the "Containers" → "Environment" section for APP_COLOR.
