Pods

Kubernetes does not deploy containers directly on the worker nodes in a cluster.

The containers are incapsulated into kubernetes objects called pods. A POD is a single instance of an application. A Pod is the smallest object that you can create in K8S.


If the user base that uses the application increases, we create additional Pods that run the application and if necessary even add an additional node(s) to the cluster to host additional pods of the same application. You do not add additional containers to an existing pod to scale applications.

Are we restricted to having a single container in a single pod? No, a single pod can have multiple containers, but they are not multiple containers of the same kind.

#depoys the nginx docker application image
kubectl run nginx

kubectl run nginx --image nginx

#see the list of pods available.

kubectl get pods
